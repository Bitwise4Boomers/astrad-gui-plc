2. Wiring & installation method
Control panel wiring
1
Use separate cable ducts inside the panel for 230V AC circuits (feeder contactor coils) and 24V DC control circuits — minimum 50mm separation or a metallic divider, per IEC 60204-1 §14.1. Never bundle high-voltage and low-voltage wiring in the same duct.
2
All 24V DC conductors: 1.0mm² minimum, ferrule-terminated (Phoenix CRIMPFOX or equivalent), colour-coded blue positive / white negative per IEC 60446. PLC I/O terminal wires should be stranded, 0.75mm² minimum, ferruled — never bare-end into PLC terminals.
3
Interposing relay coils must have a snubber diode (1N4007) across the coil — without this, the inductive back-EMF spike on PLC output turn-off can degrade or destroy the transistor output stage of the S7-1214C.
4
Motorised MCCB auxiliary contacts and contactor auxiliary contacts are brought to PLC digital inputs via potential-free (dry) contacts only — never feed an external voltage back into the PLC DI terminal. The S7-1214C sources its own 24V into the DI commons.
5
Earth/PE bus bar (25×6mm copper) runs the full length of the panel backplate with connections to every equipment chassis, DIN-rail mounting brackets, and cable screen termination points. Bond to MEB via 16mm² green-yellow conductor.
Field wiring (rooftop / array)
6
All field cables to the rooftop remote I/O station must be individually screened, twisted-pair (LIYCY or equivalent, IEC 60189) — the antenna environment at 206.5 MHz creates significant induced RF on any unscreened run. Screen drained at the PLC panel end only (single-point screening) to avoid ground loops.
7
E-STOP and fence limit-switch cables use separate, dedicated conduit or separate armoured cable from all other field wiring. Safety-loop wiring must not share conduit with motor/actuator power cables (this is mandatory per IEC 62061 SIL requirements).
8
PROFINET drop to the remote I/O unit: use Cat 6A industrial Ethernet cable (PUR-jacketed, UV-stable), with proper IP67-rated M12 D-coded connectors at the ET 200SP. Maximum cable run 100m per segment; use a Scalance switch if longer distances needed.
9
Install metallic cable trays (stainless or HDG) for all field runs; bond trays to PE at every junction. This provides additional RF shielding and lightning-current bypass path independent of cable screens.
10
TRM bank DC-DB current transducers (LEM LA 55-P): pass the DC feeder conductor through the CT aperture — do not cut the conductor. Route 4-20mA output to AI module via screened pair; screen at panel end only.
3. PLC programming — TIA Portal steps
1
Create TIA Portal project → Add S7-1214C CPU (firmware V4.5+) → add ET 200SP IM under PROFINET → configure IP addresses: PLC 192.168.1.10, remote I/O 192.168.1.11, HMI 192.168.1.20. Enable "RT (real-time)" class for PROFINET IO.
2
Define tag table (PLC tags window) before writing any code — create all I/O tags from Appendix C of the report: I0.0=52_Incomer_Aux, I0.1=Manual_Reset, Q0.0=DG_Start_Cmd, M0.0=Mains_Fail_Latch, etc. Good tag names make the ladder logic self-documenting and maintenance-friendly.
3
Rung 1 — mains-fail latch: Use NORM (normalize) and SCALE blocks to convert the AI raw count for incomer voltage to volts; add a comparator (LT block) to detect <350V; feed into a Set/Reset FF latch — only set on low-voltage condition, only reset on Manual_Reset contact. Implement hysteresis (380V reset threshold vs. 350V trip threshold) to prevent chatter.
4
Rung 3 — transfer timer: Use IEC TON block (not Siemens-proprietary S5TIME) for portability. Set PT (preset time) = T#10S. The timer bit (T1.Q) goes TRUE after 10 seconds of continuous DG_Start_Cmd = 1. Wire timer IN to Q0.0 so the timer auto-resets if DG command drops.
5
Rung 4 — interlock: The NC contact of Mains_Ctr_ON (read from I0.3, the mains contactor aux. input) must be proven open before DG_Ctr_Enable can energise. In TIA Portal, this is simply a NC contact symbol on I0.3 in series. Additionally add a MOVE block to write an alarm-level 3 entry to a data block when the transfer completes.
6
Rung 5 — OT trip: Use NORM + MUL blocks to convert 4-20mA raw count to temperature in °C. Use a GE (≥) comparator with 65°C threshold wired to a SET coil (latched). Add a separate RS latch so the trip can only be cleared by a deliberate RESET command from the HMI (operator acknowledgement), not automatically when temperature drops.
7
Watchdog / Rung 6: Enable the S7-1214C hardware watchdog under CPU properties (default 150ms). Additionally, configure a WinCC OPC-UA heartbeat write (HMI writes to a data block every 5 seconds); if the PLC does not see a fresh write within 10 seconds it sets a Comms_Fail_Flag that triggers Rung 6, resetting all outputs to safe state.
8
Alarm integration: Use the S7-1214C Alarm_DQ instruction to generate time-stamped alarm messages automatically when coil conditions change. Configure alarm buffer size ≥500 entries (CPU properties → Alarms). These flush to WinCC Alarm Management via OPC-UA, which writes to an SQLite historian on the HMI PC.
4. SCADA / HMI implementation
1
Communication protocol: OPC-UA (port 4840) between S7-1214C and WinCC Runtime — configure OPC-UA server on PLC (TIA Portal → Properties → OPC-UA → enable server, set session timeout 30s). WinCC adds PLC as OPC-UA data source. Compared to S7 protocol, OPC-UA is firewall-friendly and vendor-neutral.
2
Dashboard screen: Build power-flow mimic diagram using WinCC Graphics Designer — feeder lines change colour (green=ON, red=OFF) based on contactor aux. contact tags. Use dynamic text fields bound to AI-channel tags for live amperage readouts. Refresh rate: 1 second for status, 200ms for alarms.
3
Access levels: Configure 3 user roles in WinCC User Administrator — L1 Operator (view only + alarm acknowledge), L2 Engineer (feeder ON/OFF, DG auto/manual), L3 Maintenance (all + watchdog override + manual trip reset). All commands require a 2-step confirm dialog (click command → confirm popup with 5-second timeout).
4
Historian / trending: Enable WinCC Tag Logging for all AI channels (current, temperature, voltage) at 10-second intervals; store minimum 90 days on HMI PC local SSD. For longer retention, configure OPC-UA export to a network share on the ARIES data server. This enables correlating electrical events against ADPT observation-run records.
5
Remote monitoring: Install Siemens SINEMA Remote Connect agent on HMI PC; connect to ARIES campus VPN gateway (OpenVPN or IPsec). This allows SCADA screen viewing from the ARIES network operations room without a separate plant network. For SMS/email alerts, configure WinCC Alarm Management to trigger on Critical-severity alarms via SMTP or GSM modem.
5. Earthing & EMC requirements for the PLC installation
1
The PLC panel's PE bus must connect to the building's Main Earth Bus Bar (MEB) via a dedicated 16mm² green-yellow conductor — do not share this conductor with any other equipment's PE path. This prevents ground-loop noise from the antenna structure entering the PLC supply.
2
The ET 200SP remote I/O unit on the rooftop connects to the RF/clean signal earth (star earth system) at the nearest bonding point on the antenna ring, not the equipment/body earth run. These two earths are only bonded at the MEB — having two paths to earth creates a loop that picks up the very clutter currents the fence is designed to block.
3
All PROFINET cable screens and field-cable screens terminate at a dedicated screen/PE rail inside the PLC panel (separate from the PE bus bar) — single-point drain at the panel end. The remote field end of each screen is left floating (insulated). This is IEC 61800-3 EMC practice for inverter/RF environments.
4
Install ferrite clamp cores (Fair-Rite 2643625702, suitable for 100–300 MHz) on all field cables where they enter the PLC panel, and on the PROFINET cable at the ET 200SP port. The 206.5 MHz radar operating frequency can couple into long cable runs — the ferrites add impedance at that frequency without affecting DC or low-frequency signals.
6. Commissioning & acceptance test procedure
1
Loop check: Before energising, perform point-to-point continuity test for every DI, DO, and AI signal using a loop calibrator. Verify each tag shows correct value in the PLC tag table with signal source connected. Document with a loop-check record sheet (tag name, source, expected vs. actual reading).
2
Safety-loop test (pre-PLC-power): With PLC control power OFF, verify the hardwired E-STOP series loop independently de-energises the master contactor by pressing each E-STOP button in turn. Verify each fence-gate limit switch also breaks the loop. This test proves the safety function is truly independent of PLC — the most critical commissioning check in this design.
3
Mains-fail transfer rehearsal: Simulate a mains fail by manually opening the 52-incomer CB auxiliary-contact test terminal (I0.0 → 0). Verify Rung 1 latches M0.0, Rung 2 energises Q0.0 (DG_Start_Cmd), the 10-second TON timer begins, and Rung 4 enables DG contactor after timer Q bit goes TRUE. Verify mains contactor drops out before DG contactor picks up (interlock). Total observed transfer time should be 10–12 seconds.
4
OT trip test: Using the AI module calibration function in TIA Portal, inject a scaled value equivalent to 66°C (above 65°C threshold). Verify Rung 5 sets Q0.3 (TRM_DB_Trip_Cmd) and the alarm log records a CRITICAL latched event. Verify that simply removing the injected value does NOT auto-clear the trip — a manual HMI acknowledge is required.
5
Watchdog / comms-loss test: Unplug the PROFINET cable from the HMI PC. Verify within the watchdog timeout (10 seconds) that Rung 6 fires and all PLC outputs reset to 0 (safe state). Reconnect cable; verify SCADA reconnects automatically and logs the comms-loss event with correct timestamp.
6
Daily self-test routine: Configure a timed OB (Organisation Block OB10 in TIA Portal, scheduled 06:30 daily) to run a diagnostic cycle: check all DI feedback signals match commanded DO states, verify watchdog is live, verify safety-loop status, log a "daily self-test passed" INFO event to the alarm historian. This is visible on the SCADA alarms screen and provides an automatic maintenance audit trail.
